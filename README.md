# 🥗 NutriAI — Personalized AI Diet Recommendation System

A full-stack AI-powered nutrition assistant that generates personalized 15-day meal plans based on a user's health profile, medical conditions, allergies, and dietary preferences — with a follow-up chat interface powered by LLMs.

---

## ✨ Features

- **Personalized 15-Day Meal Plans** — Generated by LLMs based on detailed health profiles
- **Clinical Safety Rules** — Automatically adjusts meals for diabetes, hypertension, high cholesterol, thyroid disorders, lactose/gluten intolerance, and allergies
- **Multi-Provider LLM Support** — Switch between Groq (fast) and OpenRouter (flexible) at runtime
- **Conversational Chat** — Ask follow-up questions about your meal plan with full conversation memory
- **Session Management** — Save, reload, and delete multiple diet plan sessions
- **JWT Authentication** — Secure user registration and login
- **Clean Dark UI** — Built with Streamlit, custom CSS, and Google Fonts

---

## 🏗️ Architecture

```
NutriAI/
├── backend/
│   ├── main.py                  # FastAPI app — all routes and logic
│   └── providers/
│       ├── groq_provider.py     # Groq LPU inference (llama-3.3-70b)
│       └── openai_provider.py   # OpenRouter (GPT-4o-mini, etc.)
├── frontend/
│   └── app.py                   # Streamlit UI
├── .env                         # API keys (not committed)
└── requirements.txt
```

**Stack:**
- **Backend:** FastAPI + Pydantic + PyJWT + httpx
- **Frontend:** Streamlit
- **LLM Providers:** Groq API, OpenRouter API
- **Auth:** JWT (Bearer tokens)
- **Storage:** In-memory (swap with a DB for production)

## ⚡ Quickstart

### 1. Clone & create virtual environment

```bash
git clone https://github.com/yourname/nutriai.git
cd nutriai
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Set up environment variables

Create a `.env` file in the project root:

```env
GROQ_API_KEY=your_groq_api_key_here
OPENROUTER_API_KEY=your_openrouter_api_key_here
JWT_SECRET=your_random_secret_here
```

- Get a Groq API key: https://console.groq.com
- Get an OpenRouter API key: https://openrouter.ai

### 4. Run the backend

```bash
uvicorn backend.main:app --reload
```

Backend runs at `http://127.0.0.1:8000`

### 5. Run the frontend

In a separate terminal (with venv activated):

```bash
streamlit run frontend/app.py
```

Frontend runs at `http://localhost:8501`

## 🌐 API Endpoints

### Auth
| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/auth/register` | Register a new user |
| `POST` | `/auth/login` | Login and receive JWT token |

### Sessions
| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/sessions/new` | Create a new session |
| `GET` | `/sessions` | List all sessions for current user |
| `DELETE` | `/sessions/{id}` | Delete a session |

### Diet Plan
| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/diet/generate` | Generate a 15-day meal plan |

### Chat
| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/chat` | Send a message with conversation memory |
| `GET` | `/chat/history/{id}` | Get chat history for a session |

### Health
| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/health` | Server status and provider availability |

Interactive API docs available at `http://127.0.0.1:8000/docs`

## 🤖 LLM Providers

### Groq (default — recommended)
- **Diet generation model:** `llama-3.3-70b-versatile`
- **Chat model:** `llama-3.1-8b-instant`
- Ultra-fast inference via LPU hardware
- Free tier available

### OpenRouter
- **Default model:** `openai/gpt-4o-mini`
- Access to 100+ models (GPT-4o, Claude, Gemini, Mistral, etc.)
- Change model by modifying `DEFAULT_MODEL` in `openai_provider.py`

## 👤 User Profile Fields

| Field | Type | Description |
|---|---|---|
| `age` | int | User's age |
| `gender` | string | male / female / other |
| `height_cm` | float | Height in centimeters |
| `weight_kg` | float | Weight in kilograms |
| `activity_level` | string | sedentary / light / moderate / active / very_active |
| `goal` | string | lose_weight / maintain / gain_muscle |
| `diabetes` | bool | Adjusts carb distribution and GI |
| `hypertension` | bool | Limits sodium |
| `high_cholesterol` | bool | Reduces saturated fat |
| `thyroid_disorder` | bool | Noted in plan |
| `allergies` | list | Nuts, shellfish, eggs, soy, fish, sesame, peanuts, custom |
| `lactose_intolerance` | bool | Removes dairy |
| `gluten_intolerance` | bool | Removes wheat, barley, rye |
| `medications` | string | Informational |
| `diet_type` | string | regular / vegetarian / vegan / keto |
| `special_notes` | string | Any extra instructions |

## 🔒 Security Notes

> This project uses in-memory storage and plain-text passwords — suitable for development only.

For production:
- Hash passwords with `bcrypt`
- Replace in-memory dicts with PostgreSQL or MongoDB
- Set a strong random `JWT_SECRET`
- Restrict `allow_origins` in CORS middleware
- Use HTTPS
